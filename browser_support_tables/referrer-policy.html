Notes

  1. Browsers initially supported an early draft of the specification which can only use a meta tag and is only compatible with the origin value from the new spec.

  2. Chrome 21-60 did not support the same-origin, strict-origin and strict-origin-when-cross-origin values. (issue 627968)

  3. The default policy has been changed from no-referrer-when-downgrade to strict-origin-when-cross-origin. See the revised spec.

  4. Safari's ITP (on by default) downgrades _all_ cross-site subresource request (not page navigation) referrer headers to the page's origin, ignoring the unsafe-url referrer policy.
