ESLint comes with several built-in formatters to control the appearance of the linting results, and supports third-party formatters as well.

You can specify a formatter using the --format or -f flag on the command line. For example, --format json uses the json formatter.

The built-in formatter options are:

  • checkstyle
  • compact
  • html
  • jslint-xml
  • json-with-metadata
  • json
  • junit
  • stylish
  • tap
  • unix
  • visualstudio

Example Source

Examples of each formatter were created from linting fullOfProblems.js using the .eslintrc configuration shown below.

fullOfProblems.js

function addOne(i) {
    if (i != NaN) {
        return i ++
    } else {
      return
    }
};

.eslintrc:

{
    "extends": "eslint:recommended",
    "rules": {
        "consistent-return": 2,
        "indent"           : [1, 4],
        "no-else-return"   : 1,
        "semi"             : [1, "always"],
        "space-unary-ops"  : 2
    }
}
