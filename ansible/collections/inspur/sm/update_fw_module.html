inspur.sm.update_fw module – Update firmware.

Note

This module is part of the inspur.sm collection (version 2.0.0).

You might already have this collection installed if you are using the ansible package. It is not included in ansible-core. To check whether it is installed, run ansible-galaxy collection list.

To install it, use: ansible-galaxy collection install inspur.sm.

To use it in a playbook, specify: inspur.sm.update_fw.

New in version 0.1.0: of inspur.sm

Synopsis

  • Update firmware on Inspur server.

Parameters

Parameter

Comments

dual_image

string

(M5)update dual image(default) or not.

Only the M5 model supports this parameter.

Choices:

  • single
  • dual ← (default)

has_me

integer

(M5-BIOS)update me or not when update bios,only work in INTEL platform,0-no,1-yes.

Only the M5 model supports this parameter.

Choices:

  • 1

Examples

- name: Update fw test
  hosts: ism
  connection: local
  gather_facts: no
  vars:
    ism:
      host: "{{ ansible_ssh_host }}"
      username: "{{ username }}"
      password: "{{ password }}"

  tasks:

  - name: "update bios"
    inspur.sm.update_fw:
      url: "/home/wbs/SA5112M5_BIOS_4.1.8_Standard_20200117.bin"
      type: "BIOS"
      provider: "{{ ism }}"

  - name: "update bmc"
    inspur.sm.update_fw:
      url: "/home/wbs/SA5112M5_BMC_4.17.7_Standard_20200430"
      mode: "Auto"
      type: "BMC"
      dual_image: "dual"
      provider: "{{ ism }}"
