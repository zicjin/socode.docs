cisco.ise.sxp_vpns_bulk_monitor_status_info module – Information module for SXP VPNs Bulk Monitor Status

Note

This module is part of the cisco.ise collection (version 2.5.0).

You might already have this collection installed if you are using the ansible package. It is not included in ansible-core. To check whether it is installed, run ansible-galaxy collection list.

To install it, use: ansible-galaxy collection install cisco.ise.

To use it in a playbook, specify: cisco.ise.sxp_vpns_bulk_monitor_status_info.

New in version 1.0.0: of cisco.ise

  • Requirements
  • Parameters
  • Notes

Synopsis

  • Get SXP VPNs Bulk Monitor Status by id.
  • This API allows the client to monitor the bulk request.

Note

This module has a corresponding action plugin.

Requirements

The below requirements are needed on the host that executes this module.

  • ciscoisesdk >= 2.0.3
  • python >= 3.5

Notes

Note

  • SDK Method used are sxp_vpns.SxpVpns.monitor_bulk_status_sxp_vpns,
  • Paths used are get /ers/config/sxpvpns/bulk/{bulkid},
  • Supports check_mode
  • The plugin runs on the control node and does not use any ansible connection plugins, but instead the embedded connection manager from Cisco ISE SDK
  • The parameters starting with ise_ are used by the Cisco ISE Python SDK to establish the connection

Examples

- name: Get SXP VPNs Bulk Monitor Status by id
  cisco.ise.sxp_vpns_bulk_monitor_status_info:
    ise_hostname: "{{ise_hostname}}"
    ise_username: "{{ise_username}}"
    ise_password: "{{ise_password}}"
    ise_verify: "{{ise_verify}}"
    bulkid: string
  register: result
